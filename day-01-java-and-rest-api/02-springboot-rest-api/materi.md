# Materi - Spring Boot REST API

## 1. Apa Itu Spring Boot?

Spring Boot membantu developer membuat aplikasi backend Java dengan lebih cepat.

Spring Boot menyediakan:

- Embedded server, misalnya Tomcat.
- Konfigurasi awal yang lebih sederhana.
- Cara mudah membuat REST API.
- Integrasi dengan dependency seperti Spring Web.

Dengan Spring Boot, kita bisa menjalankan aplikasi backend tanpa perlu setup server manual.

## 2. Hubungan Java dengan Spring Boot

Spring Boot tidak menggantikan Java. Controller, Service, DTO, dan Model tetap berupa class Java.

Annotation seperti `@RestController`, `@Service`, dan `@RequestMapping` memberi tahu Spring bahwa class tersebut memiliki peran tertentu.

Contoh:

```java
@Service
public class CustomerService {
}
```

`CustomerService` tetap class Java biasa. Annotation `@Service` membuat Spring mengenal class tersebut sebagai service.

5. Jangan memilih Spring Boot 3.x untuk Java 8 karena Spring Boot 3 membutuhkan Java yang lebih baru.

## 8. REST API Concept

REST API adalah cara backend menyediakan endpoint agar client bisa mengirim request dan menerima response.

Format data yang umum digunakan adalah JSON.

Flow sederhana:

```text
Client/Postman -> Controller -> Service -> Response DTO -> JSON response
```

## 9. HTTP Method

| Method | Kegunaan |
| --- | --- |
| GET | Mengambil data |
| POST | Membuat data baru |
| PUT | Mengganti seluruh data |
| PATCH | Mengubah sebagian data |
| DELETE | Menghapus data |

## 10. HTTP Status Code

| Status | Arti |
| --- | --- |
| 200 OK | Request berhasil |
| 201 Created | Data baru berhasil dibuat |
| 400 Bad Request | Request tidak valid |
| 404 Not Found | Data tidak ditemukan |
| 500 Internal Server Error | Error di server |

## 11. Endpoint Naming

Gunakan nama resource, bukan nama action.

Contoh yang baik:

```text
GET /api/v1/customers
GET /api/v1/customers/{id}
POST /api/v1/customers
```

Contoh yang kurang baik:

```text
GET /getCustomerById?id=1
POST /createCustomer
```

## 12. Path Parameter

Path parameter adalah data yang menjadi bagian dari URL.

Contoh:

```text
GET /api/v1/customers/1
```

Contoh di Spring:

```java
@GetMapping("/{id}")
public CustomerResponse getCustomerById(@PathVariable Long id) {
    return customerService.getCustomerById(id);
}
```

## 13. Query Parameter

Query parameter biasanya dipakai untuk filter atau pencarian.

Contoh:

```text
GET /api/v1/customers?name=budi
```

Contoh di Spring:

```java
@GetMapping
public List<CustomerResponse> getCustomers(@RequestParam(required = false) String name) {
    return customerService.getCustomers();
}
```

## 14. Request Body

Request body digunakan untuk mengirim data yang lebih lengkap.

Contoh JSON:

```json
{
  "full_name": "Budi Santoso",
  "email": "budi@mail.com",
  "phone_number": "08123456789"
}
```

Contoh di Spring:

```java
@PostMapping
public ResponseEntity<CustomerResponse> createCustomer(@RequestBody CreateCustomerRequest request) {
    CustomerResponse response = customerService.createCustomer(request);
    return ResponseEntity.status(HttpStatus.CREATED).body(response);
}
```

## 15. snake_case JSON vs camelCase Java

JSON pada API sering menggunakan `snake_case`.

```json
{
  "full_name": "Budi Santoso"
}
```

Java menggunakan `camelCase`.

```java
private String fullName;
```

Pada Day 1, gunakan `@JsonProperty` untuk menghubungkan nama JSON dengan nama field Java.

```java
@JsonProperty("full_name")
private String fullName;
```

Dengan cara ini, client mengirim `full_name`, tetapi kode Java tetap memakai `fullName`.

## 16. Controller

Controller menerima request dan mengembalikan response.

Tugas Controller:

- Menerima HTTP request.
- Membaca parameter atau body.
- Memanggil Service.
- Mengatur HTTP status.
- Mengembalikan response.

Controller sebaiknya tipis dan tidak berisi business logic utama.

## 17. Service

Service berisi business logic.

Pada Day 1, Service bertugas:

- Menyimpan data customer ke `Map`.
- Membuat id customer.
- Mengambil data customer.
- Mengubah model menjadi response DTO.

## 18. DTO

DTO adalah Data Transfer Object. DTO digunakan untuk data yang masuk dan keluar dari API.

Jenis DTO:

- Request DTO, misalnya `CreateCustomerRequest`.
- Response DTO, misalnya `CustomerResponse`.

DTO membantu agar struktur API tidak langsung bergantung pada model internal.

## 19. Model

Model adalah object internal aplikasi.

Contoh:

```java
public class Customer {
    private Long id;
    private String fullName;
    private String email;
    private String phoneNumber;
}
```

Model tidak langsung diekspos sebagai response API agar struktur internal aplikasi tetap bisa diubah tanpa merusak kontrak API.

### Step 5: Create CustomerService

```java
package com.example.training.service;

import com.example.training.dto.CreateCustomerRequest;
import com.example.training.dto.CustomerResponse;
import com.example.training.model.Customer;
import org.springframework.stereotype.Service;

import java.util.ArrayList;
import java.util.HashMap;
import java.util.List;
import java.util.Map;

@Service
public class CustomerService {
    private Map<Long, Customer> customerStorage = new HashMap<>();
    private Long sequence = 1L;

    public CustomerResponse createCustomer(CreateCustomerRequest request) {
        Long id = sequence;
        sequence++;

        Customer customer = new Customer(
                id,
                request.getFullName(),
                request.getEmail(),
                request.getPhoneNumber()
        );

        customerStorage.put(id, customer);

        return toResponse(customer);
    }

    public CustomerResponse getCustomerById(Long id) {
        Customer customer = customerStorage.get(id);

        if (customer == null) {
            throw new RuntimeException("Customer not found");
        }

        return toResponse(customer);
    }

    public List<CustomerResponse> getCustomers() {
        List<CustomerResponse> responses = new ArrayList<>();

        for (Customer customer : customerStorage.values()) {
            responses.add(toResponse(customer));
        }

        return responses;
    }

    private CustomerResponse toResponse(Customer customer) {
        return new CustomerResponse(
                customer.getId(),
                customer.getFullName(),
                customer.getEmail(),
                customer.getPhoneNumber()
        );
    }
}
```

Untuk Day 1, jika customer tidak ditemukan, kita boleh memakai `RuntimeException` sederhana. Detail error handling akan dibahas di day berikutnya.

### Step 6: Create CustomerController

```java
package com.example.training.controller;

import com.example.training.dto.CreateCustomerRequest;
import com.example.training.dto.CustomerResponse;
import com.example.training.service.CustomerService;
import org.springframework.http.HttpStatus;
import org.springframework.http.ResponseEntity;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.PathVariable;
import org.springframework.web.bind.annotation.PostMapping;
import org.springframework.web.bind.annotation.RequestBody;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RestController;

import java.util.List;

@RestController
@RequestMapping("/api/v1/customers")
public class CustomerController {
    private final CustomerService customerService;

    public CustomerController(CustomerService customerService) {
        this.customerService = customerService;
    }

    @PostMapping
    public ResponseEntity<CustomerResponse> createCustomer(@RequestBody CreateCustomerRequest request) {
        CustomerResponse response = customerService.createCustomer(request);
        return ResponseEntity.status(HttpStatus.CREATED).body(response);
    }

    @GetMapping("/{id}")
    public ResponseEntity<CustomerResponse> getCustomerById(@PathVariable Long id) {
        CustomerResponse response = customerService.getCustomerById(id);
        return ResponseEntity.ok(response);
    }

    @GetMapping
    public ResponseEntity<List<CustomerResponse>> getCustomers() {
        List<CustomerResponse> responses = customerService.getCustomers();
        return ResponseEntity.ok(responses);
    }
}
```

## 22. Common Beginner Errors

Beberapa error yang sering terjadi:

- Port 8080 sudah digunakan aplikasi lain.
- Maven dependencies masih downloading.
- Package path tidak sesuai.
- Lupa menambahkan `@RestController`.
- Lupa menambahkan `@Service`.
- Lupa menggunakan `@RequestBody`.
- Nama field JSON tidak cocok antara `full_name` dan `fullName`.
- Salah mengetik URL endpoint.

Jika terjadi error, baca pesan error perlahan. Biasanya Spring memberi tahu class, method, atau endpoint mana yang bermasalah.
